# 命运回响

`命运回响` 是一个互动式现实人生模拟与自我觉察项目。用户会从小学、初中、高考、大学、初入职场一路走到成年后的回望，在关键人生节点中做选择。系统会根据选择记录资源状态、需求趋势、受伤线索、决策风格、随机事件和最终结局，帮助用户观察自己在压力、关系、成就、边界和修复中的反复模式。

本项目不是心理诊断工具，也不能替代心理咨询。它更接近一个文本互动游戏、心理主题叙事原型和自我反思工具。

## 项目目的

这个项目希望解决三个问题：

1. 让用户以低压力的方式体验“人生选择如何累积成模式”。
2. 帮助用户看见自己反复保护的东西，例如安全感、价值感、关系连接或掌控感。
3. 用游戏化路径、结局报告和多周目档案，让自我觉察变得更具体、更可重复体验。

项目强调“看见”，而不是“评判”。没有绝对正确的选择，不同路线会通向不同状态和结局。

## 推荐使用方式

### 方式零：本地前后端版，推荐继续开发使用

双击：

```text
一键启动前后端版.bat
```

打开后访问：

```text
http://127.0.0.1:8787
```

这个版本包含：

- Python FastAPI 后端
- 浏览器前端
- 人设初始化
- 随机事件
- 本地动态选项
- 大模型接口配置
- AI 优化人设
- AI 生成额外选择

和纯网页离线版相比，这个版本更适合接入 DeepSeek / OpenAI，因为 API Key 保存在本机后端配置文件中，不会直接写进前端 JS。

### 方式一：网页版，最适合普通用户

路径：

```text
web/index.html
```

使用方法：

1. 打开项目文件夹。
2. 进入 `web` 文件夹。
3. 双击 `index.html`。
4. 浏览器打开后即可使用。

特点：

- 不需要安装 Python。
- 不需要命令行。
- 不需要配置虚拟环境。
- 可以离线运行。
- 可以部署成真正的网址。
- 支持人设初始化、随机事件、本地动态选择和可选 AI 选择。

如果要发给完全没有计算机基础的用户，优先发 `web` 文件夹。

### 方式二：源代码版 / Streamlit 版

适合开发、调试、继续扩展剧情树。

最简单启动：

```text
双击 一键启动.bat
```

脚本会自动：

- 创建 `.venv` 本地运行环境
- 安装 `requirements.txt` 中的依赖
- 启动 Streamlit
- 打开浏览器页面

手动启动方式：

```powershell
pip install -r requirements.txt
streamlit run app.py
```

启动后浏览器会打开本地页面，通常是：

```text
http://localhost:8501
```

### 方式三：打包成 EXE

如果希望别人不安装 Python、不接触命令行，可以先在你的电脑上运行：

```text
一键打包EXE.bat
```

打包完成后，把下面这个文件夹整体发给别人：

```text
dist/命运回响
```

对方打开其中的 `命运回响.exe` 即可运行。

注意：Streamlit 本质是本地网页应用，EXE 启动后仍会打开浏览器页面，这是正常现象。

## 网页版界面操作说明

### 首页

首页提供三个主要入口：

- `开始新的人生`：进入开局人设设置。
- `继续上次进度`：读取浏览器本地保存的进度。
- 顶部 `档案`：查看保存过的终局报告。
- 顶部 `AI设置`：配置 DeepSeek 或 OpenAI 兼容接口。
- 顶部 `说明`：查看产品说明。

### 开局人设

开始新人生后，会先进入人设初始化页面。

你可以：

- 从预设人设中选择一个基础模板。
- 填写名字或代称。
- 调整开局随机性：低、中、高。
- 自定义人设描述。
- 点击 `随机生成一个人设`，由本地规则生成设定。
- 配置 AI 后，点击 `AI 优化人设`，让大模型润色人设。

人设会影响：

- 初始资源
- 初始需求
- 受伤线索倾向
- 决策风格
- 后续随机事件概率
- 部分动态选择出现的倾向

### 游戏主界面

游戏界面分为左侧剧情区和右侧状态区。

左侧包含：

- 当前章节标题
- 场景文本
- 现实锚点
- 随机状态事件
- 上一章镜像总结
- 当前可选行动
- `生成 AI 额外选择`
- `重新扰动选项`

右侧包含：

- 本局人设
- 当前资源
- 需求趋势
- 剧情路径

每次选择都会影响后续状态。部分选择只有在前面触发过某些经历后才会出现。

### 随机事件与动态选择

网页版加入了随机性：

- 每章可能出现一个随机状态事件。
- 随机事件会轻微改变资源、需求或受伤线索。
- 选项顺序可能被扰动。
- 系统可能增加一条本局随机选择。
- 用户也可以手动点击 `重新扰动选项`。

这样每一局不会完全相同，也更接近“人是会变化的”这一设定。

### AI 设置

网页版支持 OpenAI 兼容格式的大模型接口。DeepSeek 可直接使用预设。

操作：

1. 点击右上角 `AI设置`。
2. 点击 `填入 DeepSeek 预设`。
3. 填入自己的 API Key。
4. 点击保存。

DeepSeek 常用配置：

```text
接口地址：https://api.deepseek.com/chat/completions
模型名称：deepseek-chat
```

AI 功能包括：

- 开局时优化人设。
- 在人生节点中生成一条额外选择。

安全提醒：

- 不要把 API Key 写进代码。
- 不要把带有固定 Key 的网页公开发布。
- 当前网页版只把 Key 保存在本机浏览器 localStorage 中。
- 如果要公开部署，建议增加后端代理，由后端保存 Key。

常见错误：

- `HTTP 401`：Key 错误或未授权。
- `HTTP 402`：API 余额不足。
- `HTTP 429`：请求过多或限流。
- 浏览器跨域错误：纯前端直连接口受限，需要后端代理。

### 终局报告

完成所有章节后，会生成终局报告，包括：

- 结局标题
- 结局总结
- 解释文本
- 显著需求
- 受伤线索
- 决策风格摘要
- 解锁成就

可以点击 `保存到档案`，下次在档案页查看。

### 档案

档案保存于当前浏览器的 localStorage。

这意味着：

- 同一浏览器可以保留历史终局报告。
- 换浏览器或清理浏览器数据后，档案可能消失。
- 网页版不会自动上传任何用户数据。

## 源代码版界面操作说明

Streamlit 版包含：

- 首页
- 游戏页
- 终局报告页
- 档案馆
- 产品说明

主要按钮：

- `新的人生`：创建新存档。
- `继续存档`：继续上次未完成的游戏。
- `查看档案馆`：查看历史结局和成就。
- `产品说明`：查看玩法和边界。
- `返回首页`：回到首页。

Streamlit 版的数据默认保存到：

```text
life_sim_data
```

如果项目目录不可写，会自动保存到用户目录下：

```text
.life_sim_data
```

## 项目结构

```text
心里程序/
├─ app.py                 # Streamlit 主程序
├─ life_echo.py           # 源代码版模块导出入口
├─ launcher.py            # EXE/启动器入口
├─ requirements.txt       # Python 依赖
├─ pyproject.toml         # Python 项目配置
├─ 一键启动.bat            # 自动创建环境并启动 Streamlit
├─ 一键打包EXE.bat         # 使用 PyInstaller 打包
├─ 安装说明.md             # Python / EXE 打包说明
├─ README.md              # 总说明文档
└─ web/
   ├─ index.html          # 网页版入口
   ├─ style.css           # 网页版样式
   ├─ app.js              # 网页版逻辑
   ├─ story-data.js       # 从源代码导出的剧情数据
   └─ 使用说明.md          # 网页版单独说明
```

## 开发者复用

Python 中可以这样复用核心逻辑：

```python
import life_echo

state = life_echo.create_initial_state()
node = life_echo.get_current_node(state)
choices = life_echo.get_available_choices(node, state)
state = life_echo.choose(state, choices[0][0])
report = life_echo.build_final_report(state)
```

网页版剧情数据位于：

```text
web/story-data.js
```

如果要迁移到微信小程序、移动端或其他前端框架，可以优先复用这份剧情数据，再重写界面层。

## 微信小程序方向

当前项目已经有网页版本，更适合作为第一分发版本。

如果后续要做微信小程序，需要额外准备：

- 微信开发者工具
- 小程序 AppID
- 小程序页面结构
- 用户数据存储方案
- 后端代理，用于安全调用大模型 API
- 提审与发布流程

小程序不是简单复制网页文件即可上线，但可以复用剧情树、结局规则和部分界面设计。

## 产品边界

本项目不是医疗产品，不提供：

- 医学诊断
- 心理疾病判断
- 危机干预
- 心理咨询替代服务
- 对现实关系的单次结论

如果用户在体验过程中出现强烈不适、持续失眠、自伤想法或现实安全风险，应暂停使用，并寻求现实中的专业支持。
